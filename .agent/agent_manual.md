# .agent 운영 매뉴얼

이 저장소에서 작업하는 에이전트는 먼저 `.agent/`를 확인하고, 필요한 정보만 빠르게 읽은 뒤 작업을 진행한다. 이 문서의 목적은 탐색 순서, 정보 부족 시 대응 방식, 기록 규칙을 고정해 작업 속도와 일관성을 높이는 것이다.

## 기본 탐색 순서

1. `.agent/manual/Directory structure.md`를 먼저 읽고 현재 저장소 구조와 각 폴더의 목적을 파악한다.
2. 필요한 작업 범위가 정해지면 관련 폴더만 선택적으로 읽는다.
3. 루트 구현은 Python 기준으로 보고, 이전 Swift 구현은 `SwiftVision/` 아래에서 선택적으로 읽는다.
4. `SwiftVision/reference/`와 `SwiftVision/opensource/`는 역할만 빠르게 확인하고, 기본적으로 하위 전체를 순회하지 않는다.

## 폴더별 해석 기준

- `.agent/`: 에이전트 행동 기준, 질의응답 기록, 변경 이력 관리
- `.agent/skills/`: 향후 추가될 스킬 문서 보관 위치
- `.agent/manual/`: 작업 기준 문서와 운영 메모 보관 위치
- `.agent/changeLog/`: 작업 변경 이력 append 기록 위치
- `SwiftVision/`: 이전 Swift + Vision 구현 보관 위치
- `SwiftVision/reference/`: 이전 조사 문서와 참고 메모 저장소
- `SwiftVision/opensource/`: 외부 GitHub 저장소 clone 보관 위치

## 정보가 부족할 때의 행동 규칙

1. 구현에 필요한 도메인 정보가 있으면 먼저 `SwiftVision/reference/`를 확인한다.
2. 필요한 정보가 없으면 사용자에게 직접 요청하거나 `.agent/manual/QA.md`에 질문을 남긴다.
3. 후속 작업을 진행할 때는 `QA.md`에서 가장 마지막 사용자 답변을 우선 기준으로 삼는다.
4. 에이전트가 자체 판단으로 임시 결정을 내렸다면, 작업 결과나 관련 문서에 그 판단 근거가 드러나게 작성한다.

## 변경 기록 규칙

모든 파일 수정 또는 생성은 `.agent/changeLog/changeLog_{n}.md`에 append 형식으로 기록한다.

- 형식
  - `--- 변경 파일: {상대경로}`
  - `목적 : {간략한 목적}`
- 한 파일에는 최대 20개 항목까지만 기록한다.
- 20개가 차면 다음 파일인 `changeLog_2.md`, `changeLog_3.md` 순으로 이어서 기록한다.
- 기록 경로는 반드시 저장소 루트 기준 상대경로를 사용한다.

## 환경 및 의존성 관리 규칙

- 민감한 정보, API 키, 토큰, 비밀번호, 로컬 전용 설정값은 코드나 문서에 직접 쓰지 않고 `.env`에서 관리한다.
- 필요한 라이브러리와 의존성은 `requirements.txt`로 관리한다.
- 예시 코드나 작업 지시에도 민감 정보 실값을 남기지 않는다.

## 경로 작성 규칙

- 문서, 예시, 기록에는 절대경로를 사용하지 않는다.
- 경로는 누구나 재사용할 수 있도록 저장소 루트 기준 상대경로로 적는다.
- 예시: `.agent/manual/Directory structure.md`, `SwiftVision/reference/`, `SwiftVision/opensource/sample-repo/`

## 작업 목적

이 구조의 목적은 폴더 구조를 빠르게 파악하고, 필요한 부분만 읽어서 바로 작업할 수 있게 만드는 것이다. 전체 트리를 매번 처음부터 읽지 않고도 필요한 문서와 폴더에 빠르게 도달할 수 있어야 한다.
